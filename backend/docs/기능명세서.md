## 3. 상세 기능 명세

### 3.1 회원가입

- 기능명: 회원가입
- 설명: 비회원이 이메일, 비밀번호, 닉네임을 입력하여 서비스 계정을 생성한다.
- 사용자: 비회원

#### 입력값
- 이메일
- 비밀번호
- 닉네임

#### 처리 로직
1. 이메일, 비밀번호, 닉네임의 입력 여부를 확인한다.
2. 이메일 형식이 올바른지 검증한다.
3. 비밀번호 형식이 정책에 맞는지 검증한다.
4. 이메일 중복 여부를 확인한다.
5. 비밀번호를 암호화하여 저장한다.
6. 회원 정보를 데이터베이스에 저장한다.

#### 출력값
- 회원가입 성공 메시지
- 회원 ID

#### 예외사항
- 이메일 형식이 올바르지 않은 경우
- 비밀번호 형식이 정책에 맞지 않는 경우
- 닉네임이 비어 있는 경우
- 이미 가입된 이메일인 경우

---

### 3.2 로그인 / JWT 발급

- 기능명: 로그인
- 설명: 회원이 이메일과 비밀번호를 통해 로그인하고 JWT 토큰을 발급받는다.
- 사용자: 회원

#### 입력값
- 이메일
- 비밀번호

#### 처리 로직
1. 이메일로 회원 정보를 조회한다.
2. 회원이 존재하는지 확인한다.
3. 입력한 비밀번호와 저장된 암호화 비밀번호를 비교한다.
4. 인증 성공 시 Access Token과 Refresh Token을 발급한다.
5. 로그인 성공 정보를 반환한다.

#### 출력값
- Access Token
- Refresh Token
- 회원 기본 정보

#### 예외사항
- 존재하지 않는 이메일인 경우
- 비밀번호가 일치하지 않는 경우
- 탈퇴 또는 비활성화된 계정인 경우

---

### 3.3 모임 생성

- 기능명: 모임 생성
- 설명: 로그인한 회원이 새로운 모임을 생성한다.
- 사용자: 회원

#### 입력값
- 모임명
- 모임 소개
- 카테고리
- 최대 인원
- 모임 지역
- 공개 여부

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 모임명, 최대 인원 등 필수값을 검증한다.
3. 최대 인원이 허용 범위 내인지 확인한다.
4. 모임 정보를 저장한다.
5. 생성한 회원을 해당 모임의 모임장으로 등록한다.

#### 출력값
- 모임 ID
- 모임 생성 완료 메시지

#### 예외사항
- 인증되지 않은 사용자인 경우
- 필수 입력값이 누락된 경우
- 최대 인원이 허용 범위를 벗어난 경우

#### 정책
- 모임 생성자는 자동으로 모임장이 된다.
- 모임 생성자는 기본 참여 멤버에 포함된다.

---

### 3.4 모임 목록 조회

- 기능명: 모임 목록 조회
- 설명: 사용자가 전체 모임 목록을 조회한다.
- 사용자: 비회원, 회원

#### 입력값
- 카테고리(선택)
- 지역(선택)
- 키워드(선택)
- 페이지 번호(선택)

#### 처리 로직
1. 삭제되지 않은 모임 목록을 조회한다.
2. 조건이 있는 경우 카테고리, 지역, 키워드로 필터링한다.
3. 최신순 또는 정렬 기준에 따라 목록을 반환한다.

#### 출력값
- 모임 목록
- 각 모임의 요약 정보
    - 모임 ID
    - 모임명
    - 소개
    - 현재 인원
    - 최대 인원
    - 지역
    - 카테고리

#### 예외사항
- 잘못된 페이지 번호 또는 정렬 조건인 경우

---

### 3.5 모임 상세 조회

- 기능명: 모임 상세 조회
- 설명: 사용자가 특정 모임의 상세 정보를 조회한다.
- 사용자: 비회원, 회원

#### 입력값
- 모임 ID

#### 처리 로직
1. 모임 ID로 모임을 조회한다.
2. 삭제 여부를 확인한다.
3. 모임 상세 정보와 운영 정보를 반환한다.

#### 출력값
- 모임 ID
- 모임명
- 모임 소개
- 카테고리
- 지역
- 최대 인원
- 현재 인원
- 모임장 정보
- 생성일
- 공개 여부

#### 예외사항
- 존재하지 않는 모임인 경우
- 삭제된 모임인 경우

---

### 3.6 모임 수정

- 기능명: 모임 수정
- 설명: 모임장이 자신이 운영하는 모임의 정보를 수정한다.
- 사용자: 모임장

#### 입력값
- 모임 ID
- 모임명
- 모임 소개
- 카테고리
- 최대 인원
- 지역
- 공개 여부

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 모임 존재 여부를 확인한다.
3. 요청 사용자가 해당 모임의 모임장인지 확인한다.
4. 수정 가능한 항목의 유효성을 검증한다.
5. 모임 정보를 수정한다.

#### 출력값
- 수정 완료 메시지
- 수정된 모임 정보

#### 예외사항
- 인증되지 않은 사용자인 경우
- 존재하지 않는 모임인 경우
- 모임장이 아닌 경우
- 최대 인원이 현재 참여 인원보다 적은 경우

#### 정책
- 모임 수정 권한은 모임장에게만 부여된다.

---

### 3.7 모임 삭제

- 기능명: 모임 삭제
- 설명: 모임장이 자신이 운영하는 모임을 삭제한다.
- 사용자: 모임장

#### 입력값
- 모임 ID

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 모임 존재 여부를 확인한다.
3. 요청 사용자가 해당 모임의 모임장인지 확인한다.
4. 모임을 삭제하거나 소프트 삭제 처리한다.

#### 출력값
- 삭제 완료 메시지

#### 예외사항
- 인증되지 않은 사용자인 경우
- 존재하지 않는 모임인 경우
- 모임장이 아닌 경우

#### 정책
- 실제 데이터 삭제 대신 소프트 삭제를 우선 고려한다.

---

### 3.8 모임 참여 신청

- 기능명: 모임 참여 신청
- 설명: 회원이 원하는 모임에 참여를 신청한다.
- 사용자: 회원

#### 입력값
- 모임 ID

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 모임 존재 여부를 확인한다.
3. 이미 참여 중인 회원인지 확인한다.
4. 이미 신청 내역이 존재하는지 확인한다.
5. 정원이 초과되었는지 확인한다.
6. 참여 신청 정보를 저장한다.

#### 출력값
- 참여 신청 완료 메시지
- 신청 ID

#### 예외사항
- 인증되지 않은 사용자인 경우
- 존재하지 않는 모임인 경우
- 이미 참여 중인 경우
- 이미 신청한 이력이 있는 경우
- 정원이 가득 찬 경우

#### 정책
- 동일 회원은 동일 모임에 중복 신청할 수 없다.
- 이미 참여 중인 회원은 신청할 수 없다.

---

### 3.9 참여 신청 승인 / 거절

- 기능명: 참여 신청 처리
- 설명: 모임장이 참여 신청을 승인하거나 거절한다.
- 사용자: 모임장

#### 입력값
- 신청 ID
- 처리 상태(승인 / 거절)

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 신청 내역 존재 여부를 확인한다.
3. 요청 사용자가 해당 모임의 모임장인지 확인한다.
4. 이미 처리된 신청인지 확인한다.
5. 승인일 경우 모임 정원 초과 여부를 다시 확인한다.
6. 신청 상태를 승인 또는 거절로 변경한다.
7. 승인된 경우 해당 회원을 모임 참여자로 등록한다.

#### 출력값
- 처리 결과 메시지
- 변경된 신청 상태

#### 예외사항
- 인증되지 않은 사용자인 경우
- 존재하지 않는 신청 내역인 경우
- 모임장이 아닌 경우
- 이미 처리된 신청인 경우
- 승인 시 정원이 초과된 경우

#### 정책
- 승인된 회원만 정식 모임 참여자가 된다.
- 거절된 신청은 참여자로 등록되지 않는다.

### 3.11 모임 일정 조회

3. 일정 일시 기준으로 정렬하여 반환한다.

#### 출력값

### 3.12 내 모임 목록 조회

- 기능명: 내 모임 목록 조회
- 설명: 로그인한 회원이 자신이 참여 중이거나 운영 중인 모임 목록을 조회한다.
- 사용자: 회원

#### 입력값
- 회원 ID 또는 로그인 사용자 정보

#### 처리 로직
1. 로그인 여부를 확인한다.
2. 사용자가 참여 중인 모임 목록을 조회한다.
3. 사용자가 모임장인 모임과 일반 참여자인 모임을 구분하여 반환할 수 있다.

#### 출력값
- 내 모임 목록
    - 모임 ID
    - 모임명
    - 역할(모임장 / 참여자)
    - 현재 인원
    - 다음 일정 여부

#### 예외사항
- 인증되지 않은 사용자인 경우

## 4. 공통 정책

### 4.1 인증 / 인가 정책
- 회원가입, 로그인, 모임 목록 조회, 모임 상세 조회는 비회원도 접근 가능하다.
- 모임 생성, 모임 수정, 모임 삭제, 참여 신청, 내 모임 목록 조회는 로그인한 회원만 가능하다.
- 모임 수정, 모임 삭제, 참여 신청 승인/거절, 일정 생성은 모임장만 가능하다.

### 4.2 데이터 정책
- 회원 이메일은 중복될 수 없다.
- 모임 생성자는 자동으로 모임장이 된다.
- 모임 참여 신청은 중복될 수 없다.
- 승인된 사용자만 모임 참여자로 등록된다.
- 삭제 기능은 소프트 삭제를 우선 고려한다.

### 4.3 예외 처리 정책
- 잘못된 입력값은 400 Bad Request로 처리한다.
- 인증 실패는 401 Unauthorized로 처리한다.
- 권한 부족은 403 Forbidden으로 처리한다.
- 존재하지 않는 데이터는 404 Not Found로 처리한다.
- 중복 데이터는 409 Conflict로 처리한다.
